# 故事板转视频提示词格式（v2 结构化镜头格式）

故事板生成时已经存下了 9 格动作描述（`panels`）和完整生图提示词（`prompts.optimized`）。转视频提示词时复用这些文字，按以下框架合成可直接用于视频生成（Seedance 等）的提示词。

---

## 核心原则

1. **严格对齐故事板**：视频提示词必须明确标注演绎第几格到第几格，不得擅自增减动作、改变镜头顺序或偏离画面描述
2. **秒数精确**：每个时间段的秒数必须根据台词量和动作量精确计算
3. **台词嵌入**：剧本台词必须直接嵌入对应时间段，不能只概述"说了什么"
4. **分段提交**：9格故事板通常需拆成2-3段视频，每段约10-15秒

---

## 时长计算规则

| 类型 | 速度 |
|------|------|
| 中文正常语速 | **3字/秒** |
| 耳语/慢语速 | **2字/秒** |
| 内心独白VO | **2.5字/秒** |
| 纯动作/反应（无台词） | 按动作复杂度估1-4秒 |

**计算示例**：
- 台词"妈妈，爸爸藏在衣柜里30天了，是想跟我们玩捉迷藏吗？" ≈ 23字 ÷ 2字/秒（耳语）= 10秒
- VO"我丈夫出差三个月了。家里只有我和儿子。怎么可能有人藏在衣柜？" ≈ 27字 ÷ 2.5字/秒 = 11秒

---

## 分段规则

9格故事板按**情绪节拍**拆分段落，而非机械3格一段：

- 找到自然的情绪转折点（如：台词说完 → 反应开始；动作结束 → 静态凝视）
- 每段约10-15秒，不超过20秒
- 段与段之间有明确的视觉/情绪断点，方便视频生成时衔接
- 在段落描述中标注"本段演绎第X格至第Y格"

---

## 视频提示词模板（v2 结构化镜头格式）

每段视频提示词使用以下固定结构：

**段头（全局声明，每段写一次）**：
```
{{Portrait 1}} 为[角色A名] {{Portrait 2}} 为[角色B名] {{Portrait N}} 为故事板

真人电影风格，画面不要出现任何文字，生成视频无BGM。本段演绎[故事板名]第X格至第Y格。严格按照提示词与故事板演绎剧情，每一秒的景别、动作、情绪必须与提示词完全一致。整体质感：真人毛孔清晰可见，皮肤纹理真实细腻，发丝自然带飞散，[面料]纹理可见，自然肤色无磨皮，面部微表情生动。[光照描述]。
```

**每个镜头单元（重复N次）**：
```
c{编号},{时长}s,（第{格数}格，{shot}）
(空间:{场景名})
(姿态:{{角色A}}-{动作基底}; {{角色B}}-{动作基底})
(位置:{{角色A}}-{画面中的位置}; {{角色B}}-{画面中的位置})
(情绪:{{角色A}}-{情绪词} {强度}/10, {{角色B}}-{情绪词} {强度}/10)
{散文段：景别+镜头运动+光照+身体反应链+台词}【不出现任何文字字幕】
```

**{{Portrait N}} 映射规则**：
- {{Portrait 1}} = 故事板参考图中的第一个角色
- {{Portrait 2}} = 第二个角色（如有）
- 最后一个 {{Portrait}} 始终为"故事板"（即本故事板的主图）
- 参考图来源 = 故事板 meta.json 中的 image_refs 字段

---

## 镜头对齐规则（关键）

故事板每格的 `panels` 已包含 `shot` 字段（如 "extreme close-up"、"medium shot"、"wide shot"）。视频提示词**必须直接引用**该镜头信息，不得自行编造或替换镜头。

**对齐方式**：
- 每个镜头单元标题写：`c{N},{时长}s,（第{格数}格，{shot}）`
- shot 是**硬约束**：故事板说 extreme close-up 就必须是极特写，说 wide shot 就必须是全景
- 动作/台词可微调，镜头不能换

**错误示例**：
```
c07,5s,（第七格，medium shot）
(空间:小乐卧室)
特写，婉瑜僵硬地慢慢转头...  ← shot是"medium shot"，散文写了"特写"，不一致！
```

**正确示例**：
```
c07,5s,（第七格，medium shot）
(空间:小乐卧室)
中景，婉瑜僵硬地慢慢转头...  ← 与shot一致
```

---

## 每个镜头单元必须包含

1. **标题行**：`c{N},{时长}s,（第{格数}格，{shot}）` — 镜头自带时长，独立单元
2. **空间字段**：`(空间:{场景名})` — 场景定位
3. **姿态字段**：`(姿态:{{角色A}}-{动作}; {{角色B}}-{动作})` — 双角色动作基底，来自panels.action
4. **位置字段**：`(位置:{{角色A}}-{位置}; {{角色B}}-{位置})` — 双角色在画面中的位置，**强制字段，防止角色消失**
5. **情绪字段**：`(情绪:{{角色A}}-{情绪词} X/10, {{角色B}}-{情绪词} X/10)` — 情绪强度量化，控制递进节奏
6. **散文段**：景别+镜头运动+光照+身体反应链（喉结、指节、肩膀等）+台词（用`<>`包裹）

**情绪强度参考**：
- 2-3/10：平静、温柔、安心
- 5-6/10：震惊、紧张、警觉
- 7-8/10：恐惧、愤怒、崩溃
- 9-10/10：极度恐惧、歇斯底里

**台词语法**：用 `<>` 包裹，并标注语速，如：`{{小乐}}用中文耳语说道<妈妈，爸爸藏在衣柜里30天了>`

## 工作流步骤

```
用户：把"冷开场"转成视频提示词

Skill：
1. 读取 storyboards/SB-E1-00冷开场/meta.json
2. 取出 panels（9格，每格含 shot + action）+ prompts.optimized（生图提示词基底）
3. 从用户提供的剧本中提取本段台词
4. 计算每句台词时长（3字/秒，耳语2字/秒，VO 2.5字/秒）
5. 按情绪节拍拆分9格为2-3段
6. 按 v2 结构化镜头格式填写：
   a. 段头（全局声明：Portrait映射 + 风格 + 质感 + 光照）
   b. 每格一个镜头单元：c{N},{时长}s + 四字段头（空间/姿态/位置/情绪）+ 散文段
   c. 每个角色名在字段中用 {{角色名}} 变量形式引用
   d. 台词用 <> 包裹，并标注语速
7. 展示给用户确认，可修改
8. 确认后：
   a. 保存到 meta.json 的 video_prompts 字段（数组，每段一个元素）
   b. 输出可直接复制的提示词文本
```

## meta.json 字段（故事板专用）

```json
{
  "panels": [
    {"index": 1, "shot": "extreme close-up", "action": "小乐的脸侧过来，嘴唇凑近婉瑜的耳朵"},
    {"index": 2, "shot": "close-up", "action": "小乐在婉瑜耳边小声说话，嘴唇微动"},
    ...
  ],
  "video_prompts": [
    {
      "part": 1,
      "title": "冷开场 Part 1：耳语→书落地",
      "panels": "1-5",
      "duration_seconds": 15,
      "prompt": "{{Portrait 1}} 为婉瑜...\n\n0-2秒：第一格（extreme close-up）\n极特写..."
    },
    {
      "part": 2,
      "title": "冷开场 Part 2：恐惧渗入",
      "panels": "6-9",
      "duration_seconds": 15,
      "prompt": "{{Portrait 1}} 为婉瑜...\n\n0-4秒：第六格（close-up）\n特写..."
    }
  ]
}
```

**注意**：`video_prompt`（旧字段，单字符串）已升级为 `video_prompts`（新字段，数组），支持多段提交。兼容旧数据：读取时优先用 `video_prompts`，如不存在则回退 `video_prompt`。

**panels 写入时机**：生成故事板时，把模板中拆解的 9 格动作同步写入 meta.json。转视频提示词时直接读取，无需用户重新描述。
